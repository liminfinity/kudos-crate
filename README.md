## 🌍 О проекте

**МИРА** — это корпоративная HR-платформа для сбора обратной связи, взаимного признания (kudos), проведения опросов, 360-градусных ревью и аналитики командного настроения. Платформа создана для компаний, которые хотят выстроить культуру открытости и конструктивного диалога внутри команд.

### Какую проблему решает

- **Разрозненность обратной связи.** В большинстве компаний фидбэк не фиксируется, теряется и не анализируется. МИРА собирает все взаимодействия в единую систему.
- **Отсутствие объективной аналитики.** Руководители и HR-специалисты принимают решения интуитивно. МИРА предоставляет данные: индексы удовлетворённости, вовлечённости, тренды настроения.
- **Токсичность и конфликты.** Модуль AI-модерации автоматически смягчает агрессивные формулировки, сохраняя смысл.
- **Формальность признания.** Система благодарностей (Kudos) позволяет отмечать вклад коллег в режиме реального времени.

### Для кого подходит

- IT-компании (10–500 сотрудников)
- HR-отделы, которые хотят оцифровать обратную связь
- Руководители, которым важно держать руку на пульсе команды
- Организации с удалёнными и гибридными командами

## ✨ Ключевые возможности

| Модуль | Описание |
|--------|----------|
| **Обратная связь** | Структурированные отзывы с привязкой к рабочему эпизоду, подкатегориями и AI-модерацией |
| **Благодарности (Kudos)** | 6 категорий признания вклада коллег |
| **Отзыв 180°** | Развёрнутая полугодовая оценка коллеги с указанием сильных/слабых сторон |
| **Review 360°** | Полноценные циклы 360-градусной оценки с автоматическим распределением ревьюеров |
| **Полугодовые опросы** | Многошаговые wizard-опросы с эмодзи-шкалой, Likert-карточками и текстовыми полями |
| **Дневник руководителя** | Инструмент для фиксации наблюдений за командой |
| **Embed-формы** | Встраиваемые виджеты для сбора обратной связи без авторизации |
| **Состояние атмосферы (CSI)** | Индекс настроения компании: (Позитивные − Негативные) / Всего |
| **Индекс удовлетворённости** | Доля позитивных отзывов: (Позитивные / Всего) × 100% |
| **Индекс вовлечённости** | Взвешенная метрика: feedback × 0.5 + kudos × 0.2 + surveys × 0.3 |
| **Тепловая карта взаимодействий** | Визуализация частоты и тональности обмена отзывами |
| **Граф отношений** | Сетевая карта коммуникаций в компании |
| **Серьёзные сигналы** | Автоматическое отслеживание критических инцидентов |
| **AI-рекомендации** | Персональные рекомендации по развитию на основе агрегированных данных |
| **Ассистент МИРА** | Контекстный AI-чат, привязанный к текущей странице и роли пользователя |

## 🐳 Запуск через Docker

### Разработка (с hot reload)

```bash
# 1. Скопировать переменные
cp .env.example .env
# 2. Заполнить .env

# 3. Запустить контейнеры
docker compose up --build

# Приложение: http://localhost:5173
# PostgreSQL: localhost:5432
```

Контейнеры:
- **app** — Vite dev-сервер с hot reload, монтирование исходников
- **db** — PostgreSQL 16 Alpine с persistable volume

### Production

```bash
# 1. Настроить production-переменные
cp .env.example .env.production
# 2. Заполнить .env.production

# 3. Запустить
docker compose -f docker-compose.prod.yml up --build -d

# Приложение: http://localhost (Nginx → порт 80)
# Health: http://localhost:3000/health
```

Контейнеры:
- **app** — Multi-stage build: `node:20-alpine` → `serve` (статика, порт 3000)
- **nginx** — Reverse proxy (порт 80 → app:3000)
- **db** — PostgreSQL 16 с healthcheck

### Полезные команды

```bash
# Остановить
docker compose down

# Пересобрать без кэша
docker compose build --no-cache

# Логи
docker compose logs -f app

# Проверка здоровья
curl http://localhost:3000/health
```

### Dockerfile — multi-stage build

```
Stage 1 (deps)    → npm ci (установка зависимостей)
Stage 2 (builder) → npm run build (сборка Vite)
Stage 3 (runner)  → serve -s dist (раздача статики)
```

## 🌱 Демо-данные и seed

Платформа включает edge function `seed-data`, которая генерирует полноценный набор демо-данных для тестирования:

### Что создаётся

| Сущность | Количество | Описание |
|---------|-----------|----------|
| **Пользователи** | 36 | 1 admin, 2 HR, 3 менеджера, 30 сотрудников |
| **Команды** | 5 | Backend, Frontend, Product, DevOps, QA |
| **Подкатегории** | 12 | 6 позитивных + 6 негативных |
| **Рабочие эпизоды** | 25 | Спринты, ретро, релизы, хакатон |
| **Обратная связь** | ~200+ | С подкатегориями, тональностью, архетипами сотрудников |
| **Благодарности** | ~80+ | 6 категорий, рандомные комментарии |
| **Отзывы 180°** | 60 | Сильные/слабые стороны, баллы коллаборации |
| **Циклы 360°** | 2 | 1 закрытый + 1 активный (published) |
| **Назначения 360°** | ~120 | По 3 ревьюера на участника, частично заполнены |
| **Полугодовые опросы** | 2 | 1 закрытый + 1 открытый |
| **Назначения опросов** | ~50 | Частично заполнены ответами |

### Архетипы сотрудников

Seed создаёт реалистичное распределение:
- **15% «Конфликтные»** — преобладание негативных отзывов (20% позитивных)
- **15% «Звёзды»** — почти все отзывы позитивные (95%)
- **15% «Активные-негативные»** — много активности, но 40% позитива
- **55% «Нормальные»** — стандартный баланс (70% позитивных)

## 📖 Полное описание работы системы

### 1. Обратная связь (Feedback)

#### Обычный отзыв

Пользователь оставляет структурированный отзыв о коллеге:

1. **Выбор получателя** — из списка активных сотрудников
2. **Привязка к эпизоду** — рабочий период (спринт, ретро, релиз)
3. **Тональность** — позитивная или негативная
4. **Подкатегории** — множественный выбор из справочника:
   - Позитивные: «Помог разобраться», «Быстро ответил», «Проявил инициативу», «Качественно сделал», «Командная работа», «Менторство»
   - Негативные: «Игнорировал сообщения», «Сорвал сроки», «Не держал в курсе», «Низкое качество», «Конфликтность», «Некомандный игрок»
5. **Текст** — свободный комментарий, проходит AI-модерацию
6. **Серьёзный инцидент** — чекбокс (только для негативных), попадает в раздел «Сигналы»

Текст автоматически обрабатывается edge function `process-text`:
- Удаление мата и оскорблений
- Исправление орфографии и пунктуации
- Смягчение агрессивных формулировок (сохраняя смысл)
- Расчёт `toxicity_score` (0–1)
- Детекция бессмысленного текста (status: `INVALID`)

#### Видимость отзывов

- Автор видит свои отправленные отзывы
- Менеджер, HR, Admin видят все отзывы
- Сотрудник **не видит** отзывов о себе напрямую (только через аналитику)

### 2. Благодарности (Kudos)

Лёгкий формат признания вклада коллеги:

| Категория | Описание |
|-----------|----------|
| `helped_understand` | Помог разобраться |
| `emotional_support` | Эмоциональная поддержка |
| `saved_deadline` | Спас дедлайн |
| `shared_expertise` | Поделился экспертизой |
| `mentoring` | Менторство |
| `team_support` | Командная поддержка |

- Комментарий необязательный, также проходит AI-модерацию
- Все авторизованные пользователи видят все благодарности
- **Help Index** = кол-во благодарностей сотрудника / среднее по компании

### 3. Отзыв 180°

Полугодовая развёрнутая оценка коллеги:

- **Период** — выбор (H1-2025, H2-2025, H1-2026 и т.д.)
- **Сильные стороны** — множественный выбор из списка
- **Слабые стороны** — множественный выбор из списка
- **Развёрнутый комментарий** — с AI-модерацией
- **Балл коллаборации** — числовая оценка (1–5)

Видимость:
- Автор видит свои отправленные 180°
- Менеджер, HR, Admin видят все 180°
- Аналитика доступна только менеджерам и выше

### 4. Review 360°

Полноценная система 360-градусной оценки:

#### Жизненный цикл

1. **Создание цикла** (менеджер/HR/admin) — название, год, даты открытия и закрытия
2. **Генерация назначений** — edge function `generate-360-assignments`:
   - Параметры: `required_reviews_per_user` (по умолчанию 3), `max_assignments_per_reviewer` (по умолчанию 4)
   - Алгоритм: случайное распределение с учётом ограничений
3. **Публикация** — статус `published`, назначения становятся видны ревьюерам
4. **Заполнение** — каждый ревьюер заполняет анкету для каждого назначенного ревьюи
5. **Закрытие** — статус `closed`, доступна аналитика

#### Ответы 360°

JSON-структура:
```json
{
  "strengths": "Коммуникабельность",
  "weaknesses": "Тайм-менеджмент",
  "rating": 4,
  "comment": "Отличный коллега"
}
```

#### Видимость

- Ревьюер видит свои назначения и может заполнить ответы
- Менеджер/HR/Admin видят все циклы, назначения и ответы
- Сотрудник **не видит** ответов о себе

### 5. Опросы

#### Полугодовой опрос

Многошаговый wizard:
- Шаг 1–N: вопросы из шаблона (`survey_templates.schema_json`)
- Эмодзи-шкала (1–5)
- Likert-карточки
- Текстовые поля
- Прогресс-бар

Жизненный цикл:
1. Создание цикла (`survey_cycles`) с привязкой к шаблону
2. Назначение сотрудникам (`survey_assignments`)
3. Заполнение (статусы: `not_started` → `in_progress` → `submitted`)
4. Закрытие цикла, аналитика

#### Дневник руководителя

Отдельный тип опроса для фиксации наблюдений руководителя о команде. Доступен только менеджерам и выше.

#### Embed-опросы

Встраиваемые формы для сбора обратной связи без авторизации:
- `/embed/survey/:cycleId` — опрос
- `/embed/feedback` — отзыв
- `/embed/kudos` — благодарность

Ответы сохраняются в таблицу `embed_responses` с metadata.

### 6. Аналитика

#### Satisfaction Index (Индекс удовлетворённости)

```
SI = (Позитивные отзывы / Общее кол-во) × 100%
```

- Фильтры: период, команда, сотрудник
- Гистограмма по сотрудникам:
  - ≥75% → зелёный
  - 50–75% → нейтральный
  - <50% → красный

#### Engagement Index (Индекс вовлечённости)

```
EI = feedback_activity × 0.5 + kudos_activity × 0.2 + survey_completion × 0.3
```

- Scatter plot: X=вовлечённость, Y=удовлетворённость, размер точки=кол-во полученных отзывов
- Типология сотрудников:
  - 🟢 Активный + позитивный
  - 🟡 Активный + конфликтный
  - 🔵 «Тихая звезда»
  - ⚪ Пассивный

#### Company Mood Index (CSI)

```
CSI = (Позитивные − Негативные) / Всего
```

Диапазон: от -1 (все негативные) до +1 (все позитивные). Включает временную линию тренда.

#### Тепловая карта

Визуализация частоты и тональности обмена отзывами между сотрудниками. Фильтры по команде и периоду.

#### Граф связей

Сетевая карта коммуникаций: узлы — сотрудники, рёбра — отзывы. Цвет и толщина рёбер отражают тональность и частоту.

### 7. AI-интеграция (МИРА)

#### Модерация текста (`process-text`)

5 контекстов модерации:
- **feedback** — стандартная модерация отзывов
- **critical** — модерация серьёзных инцидентов (не смягчает обвинения)
- **kudos** — лёгкая редактура благодарностей
- **survey** — структурирование ответов опросов
- **manager-diary** — модерация дневника руководителя

Возвращает:
```json
{
  "status": "OK",
  "processed_text": "Исправленный текст",
  "toxicity_score": 0.15,
  "was_modified": true
}
```

#### AI-рекомендации (`recommendations`)

3 уровня рекомендаций:
- **employee** — персональные рекомендации по развитию
- **team** — рекомендации для руководителя команды
- **company** — стратегические рекомендации для HR

На вход принимает агрегированные метрики, генерирует 3–5 конкретных рекомендаций.

#### Контекстный ассистент (`assistant`)

- Streaming-ответы (SSE)
- Контекст текущей страницы (25 страниц)
- Ограничения по роли (employee не видит аналитику)
- Модель: `google/gemini-3-flash-preview`

### Реализация RBAC

- **Frontend:** `ProtectedRoute` с параметром `allowedRoles` + фильтрация навигации по роли

- **Edge Functions:** Проверка JWT-токена и роли в заголовке `Authorization`

Платформа использует Edge Functions (Deno) для серверной логики:

| Функция | Метод | Описание | Аутентификация |
|---------|-------|----------|---------------|
| `/process-text` | POST | AI-модерация текста | Нет (public) |
| `/assistant` | POST | AI-ассистент (streaming SSE) | Нет (public) |
| `/recommendations` | POST | AI-рекомендации по развитию | Нет (public) |
| `/generate-360-assignments` | POST | Генерация назначений 360° | Нет (public) |

| `/seed-data` | POST | Генерация демо-данных | Service Role Key |

### Примеры запросов

#### Модерация текста

```bash
curl -X POST https://your-project.supabase.co/functions/v1/process-text \
```

#### AI-рекомендации

```bash
curl -X POST https://your-project.supabase.co/functions/v1/recommendations \
```

## 🔒 Безопасность

### Аутентификация

- **JWT-токены** — выдаются при входе, проверяются на каждый запрос
- **Пароли** — хранятся в хешированном виде (bcrypt, обрабатывается платформой аутентификации)
- **Сессии** — автоматическое обновление через `onAuthStateChange`

### Row-Level Security (RLS)

Каждая таблица защищена RLS-политиками:

- `feedback` — автор видит свои, менеджеры+ видят все
- `kudos` — видят все авторизованные, создавать может только автор
- `feedback_180` — автор видит свои, менеджеры+ видят все
- `review_360_*` — ревьюеры видят свои назначения, менеджеры+ видят все
- `survey_*` — пользователи видят свои назначения, admin управляет
- `profiles` — читают все авторизованные, редактирует только владелец
- `user_roles` — читает владелец и admin, управляет admin
- `teams` — читают все, управляет admin

### Функции безопасности БД

```sql
has_role(_user_id uuid, _role app_role) → boolean
is_manager_or_above(_user_id uuid) → boolean
get_user_role(_user_id uuid) → app_role
```

### Embed-формы

- Публичные формы (`/embed/*`) работают без авторизации
- Вставка в `embed_responses` разрешена всем (RLS: `WITH CHECK (true)`)
- Чтение embed-ответов — только менеджеры+

### Healthcheck

```
GET /health → 200 OK
```

Используется Docker HEALTHCHECK для мониторинга доступности.

## 📝 Заключение

**МИРА** — это комплексная платформа корпоративной обратной связи, спроектированная для создания культуры открытости и конструктивного диалога в командах.

### Что отличает МИРА

- **Полный цикл обратной связи** — от быстрой благодарности до глубокого 360° ревью
- **AI-модерация** — автоматическое смягчение токсичности с сохранением смысла
- **Аналитика на основе данных** — 10+ дашбордов с ключевыми метриками HR
- **Гибкие роли** — чёткое разграничение доступа (employee → manager → HR → admin)
- **Контекстный AI-ассистент** — помощь прямо на странице, адаптированная под роль
- **Embed-интеграция** — сбор обратной связи без авторизации
- **Production-ready** — Docker, healthcheck, RLS, multi-stage build

### Для кого

- HR-отделы, которые хотят оцифровать обратную связь и видеть тренды
- Руководители, которым нужны объективные метрики о настроении команды
- IT-компании с 10–500 сотрудниками
- Организации, строящие культуру прозрачной обратной связи
